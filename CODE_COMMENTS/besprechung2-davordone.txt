-Unity ist client, hat nen asynchronen sendtopython und askforpython-thread, in dem es alle x sekunden python nen statusupdate gibt mit einem vektor seiner informationen. Ausserdem fragt es alle y sekunden python nach neuen ANN-werten, die ein asynchroner thread in eine variable mit timestamp schreibt. Jedes Frame guckt er ob er schon python schon neue infos geben mochte und ob er neue infos abfragen mochte, und nutzt den jeweils aktuellsten wert, falls nicht zu alt.
-python ist der server. dafur hat es 2 main threads mit listeneports, an einem port meldet unity sich um ihm vektoren zu schicken, diese akzeptiert python und updated einen globalen value ("vektor") mit den neuen werten+timestamp. Am anderem port meldet sich unity wenn es nach neuen ANNresults fragt, dann schickt python ihm die neuesten. Ferner hat es einen dritten thread, der alle sich alle z sekunden den vektor anschaut und, nur falls noch nicht passiert, ein ANN in einem neuem thread simuliert, dass die globale variable ANNresults updated (aber auch nur, falls dieses network nicht von einem spater gestartetem uberholt wurde, dafur guckt es sich timestamps an)
